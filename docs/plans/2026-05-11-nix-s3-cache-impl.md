# Nix S3 Cache Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a publicly-readable Yandex.Cloud S3 bucket as a Nix binary cache with a manual-dispatch GitHub Action to build and upload packages.

**Architecture:** `common/cache.nix` module configures the S3 substituter for all NixOS machines. A new `build-cache.yml` workflow lets the user manually trigger builds of arbitrary packages and uploads signed closures to S3.

**Tech Stack:** Nix, NixOS modules, GitHub Actions, Yandex.Cloud Object Storage (S3-compatible)

```nix
}
```

### Task 3: Create GitHub Action workflow

**Files:**
- Create: `.github/workflows/build-cache.yml`

**Step 1: Create the workflow**

```yaml
name: Build and Cache Package

on:
  workflow_dispatch:
    inputs:
      package:
        description: "Package to build and cache (e.g. nixpkgs#telemt)"
        required: true
        type: string

jobs:
  build-and-cache:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: cachix/install-nix-action@v25
        with:
          nix_path: nixpkgs=channel:nixos-unstable

      - name: Set up dummy secrets
        run: make setup-dummy-secrets

      - name: Build package
        run: nix build "${{ inputs.package }}"

      - name: Sign and upload to S3 cache
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.YC_S3_ACCESS_KEY }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.YC_S3_SECRET_KEY }}
        run: |
          echo "${{ secrets.NIX_CACHE_PRIVATE_KEY }}" > /tmp/cache-private-key.pem
          nix copy \
            --to 's3://nix-cache?region=ru-central1&scheme=https&endpoint=storage.yandexcloud.net' \
            --option secret-key-files /tmp/cache-private-key.pem \
            $(nix path-info "${{ inputs.package }}" --closure)
          rm -f /tmp/cache-private-key.pem
```

**Step 2: Verify YAML is valid**

Run: `python3 -c "import yaml; yaml.safe_load(open('.github/workflows/build-cache.yml'))"`
Expected: No errors

**Step 3: Commit**

```bash
git add .github/workflows/build-cache.yml
git commit -m "feat: add build-cache workflow for S3 binary cache"
```

### Manual Steps (Outside This Plan)

These must be done by the user before the workflow can actually run:

1. **Generate signing keys:**
   ```bash
   nix-store --generate-binary-cache-key nix-cache-1 \
     ./cache-private-key.pem ./cache-public-key.pem
   ```

2. **Update `common/cache.nix`** with the real public key from `cache-public-key.pem`

3. **Create Yandex.Cloud service account** with `storage.uploader` + `storage.viewer` roles

4. **Create static access key** for the service account

5. **Configure GitHub Secrets:**
   - `YC_S3_ACCESS_KEY` — static access key ID
   - `YC_S3_SECRET_KEY` — static secret key
   - `NIX_CACHE_PRIVATE_KEY` — full contents of `cache-private-key.pem`

6. **Set bucket to public read** (bucket policy allowing `s3:GetObject` + `s3:GetBucketLocation` for `Principal: *`)

7. **Initialize bucket** — upload `nix-cache-info` file:
   ```
   StoreDir: /nix/store
   WantMassQuery: 1
   Priority: 40
   ```
   ```bash
   aws --endpoint-url=https://storage.yandexcloud.net \
       --region ru-central1 \
       s3 cp nix-cache-info s3://nix-cache/nix-cache-info
   ```
