# Coding Agents Global Instructions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a home-manager module that exposes a single `home/coding-agents/AGENTS.md` source file as symlinks at `~/.claude/CLAUDE.md` and `~/.config/opencode/AGENTS.md`, gated by per-target enable flags.

**Architecture:** New module `home/coding-agents/{default.nix,AGENTS.md}` declares two `mkEnableOption` flags (`codingAgents.claude.enable`, `codingAgents.opencode.enable`). Each gated `home.file.<path>` entry uses the same source file. The module is wired into `home/default.nix`'s `imports`. Per-host opt-in lives in `flake.nix` host configs alongside the existing `software.*.enable = true;` lines.

**Tech Stack:** Nix flakes, home-manager (release-25.11), `nixfmt-rfc-style`.

**Spec:** `docs/superpowers/specs/2026-04-26-coding-agents-md-design.md`

## Task 2: Create the home-manager module

**Files:**
- Create: `home/coding-agents/default.nix`

- [ ] **Step 1: Write the module**

Write to `home/coding-agents/default.nix` exactly:

```nix
{ lib, config, ... }:

let
  cfg = config.codingAgents;
in
{
  options.codingAgents = {
    claude.enable = lib.mkEnableOption "global CLAUDE.md for Claude Code";
    opencode.enable = lib.mkEnableOption "global AGENTS.md for opencode";
  };

  config = {
    home.file.".claude/CLAUDE.md" = lib.mkIf cfg.claude.enable {
      source = ./AGENTS.md;
    };
    home.file.".config/opencode/AGENTS.md" = lib.mkIf cfg.opencode.enable {
      source = ./AGENTS.md;
    };
  };
}
```

Run: `nix develop --command nixfmt home/coding-agents/default.nix`
Expected: exits 0, no diff (file already follows nixfmt-rfc-style).

If nixfmt rewrites it, accept the rewrite — the project rule is `nixfmt-rfc-style` everywhere.
